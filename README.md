# 9회말 연구소

실존 선수, 구단 로고, 공식 경기 기록 없이 야구 팬의 감정과 상황을 공유 카드로 만드는 비공식 팬메이드 웹앱입니다.

## 타겟

- 직관 전후에 짧은 문구를 공유하고 싶은 야구 팬
- 팀·선수 실명 없이도 야구 감성을 즐기고 싶은 라이트 팬
- 친구 단톡방, 스토리, 커뮤니티에 올릴 안전한 밈 카드가 필요한 사용자

## 핵심 기능

- 상황과 톤을 고르면 야구 감정 밈 카드 생성
- 첫 방문자가 버튼 하나로 시작하는 `10초 관전 타입` 카드
- 커뮤니티에 바로 올리기 쉬운 `라이브 짤판`
- 앱 안의 반응 선택과 베이스 샘플을 보여주는 `팬심 퍼센테이지`
- 최근 만든 카드를 이 브라우저에만 저장하는 `보관함`
- 보관함 기본 최근 3장 보기와 전체 보기 토글
- 모바일 공유 시트 또는 카드 상태를 그대로 여는 `공유 링크`
- 커뮤니티 게시글에 붙이기 쉬운 안전 고지 포함 `캡션 복사`
- 서버 없이 질문 링크로 이어지는 `단톡방 반응 릴레이`
- 질문 링크를 받은 사람이 바로 답장 카드 흐름을 이해하는 릴레이 답장 모드
- 미리보기에서 바로 누르는 `친구에게 묻기` 릴레이 진입점
- 날짜마다 바뀌는 `오늘의 덕아웃` 콘텐츠
- 오늘의 한 줄, 단톡방 질문, 안전 공유 소재를 바로 카드로 만드는 원탭 콘텐츠 카드
- 오늘 만든 카드 수, 주간 결산까지 남은 장수, 결산 버튼을 보여주는 `오늘의 1장 챌린지`
- 브라우저 안에만 저장되는 `덕아웃 출석` 연속 일수
- 보관함 카드 3장 이상으로 만드는 `주간 감정 결산`
- 삭제한 보관함 카드를 바로 복구하는 `되돌리기`
- 운영자에게 보낼 문구를 만드는 `권리·안전 검토 요청`
- 사용성 의견을 서버 없이 전달하는 `베타 피드백` 문구 복사
- 저장 상태를 보여주고 지울 수 있는 `내 데이터` 관리
- 정적 공개 배포를 위한 `개인정보 안내`와 `이용·권리 기준` 페이지
- 모바일 하단 탭과 PWA 기본 설정
- 정사각, 스토리, 가로형 PNG 저장
- 미리보기 아래에서 바로 공유, 링크 복사, PNG 저장
- 선수 대신 일상 루틴을 배치하는 `내 인생 타순`
- 비공식 팬메이드, 창작 상황, 공식 기록 아님 고지

## 운영 원칙

- 실존 선수명, 구단명, 로고, 유니폼, 중계 캡처, 공식 기록을 사용하지 않습니다.
- 앱 콘텐츠는 가상 상황, 팬 감정, 일상 은유, 창작 문구로만 구성합니다.
- 질문 릴레이와 감정 결산은 실제 경기 집계나 공개 리더보드가 아닌 개인 브라우저 기준 카드입니다.
- 상대 팬덤, 선수, 심판을 공격하거나 부상·사생활을 추측하는 표현은 피합니다.
- 권리 침해 요청이 들어오면 해당 콘텐츠를 숨김 처리하고 운영자가 검토하는 흐름을 둡니다.
- 보관함, 반응 카운트, 덕아웃 출석은 서버 전송 없이 사용자 브라우저 안에만 저장합니다.
- 사용자는 `내 데이터`에서 브라우저 안의 보관함, 반응 카운트, 출석 기록을 지울 수 있습니다.
- 검토 요청 기능은 서버 접수가 아니라 사용자가 운영자에게 전달할 문구를 복사하는 MVP 기능입니다.
- 베타 피드백 기능도 서버 접수가 아니라 사용자가 직접 전달할 문구를 복사하는 MVP 기능입니다.
- `legal/privacy.html`은 현재 MVP가 수집하지 않는 정보, 로컬 저장소, 공유 링크, 캐시 동작을 설명합니다.
- `legal/terms.html`은 허용 콘텐츠, 금지 콘텐츠, 팬심 퍼센테이지의 비공식 성격, 향후 상업 기능 기준을 설명합니다.

## 준비 목록

- 수익화는 당장 제품 화면에 넣지 않습니다.
- 후보 1: 응원용품 제작, 글러브 수선, 사회인야구 장비샵 같은 업체 홍보 탭.
- 후보 2: 야구용품 중고거래는 `장비장터 Beta`로 수요를 확인한 뒤 안전결제와 분쟁 대응 기준이 준비되면 검토.
- 어떤 경우에도 구단 로고, 선수명, 공식 굿즈처럼 보이는 권리물 제작·거래는 허용하지 않습니다.

## 무료 웹 베타

- 이 앱은 정적 파일만으로 동작하므로 무료 정적 호스팅에서 작은 공개 베타를 시작할 수 있습니다.
- 추천 순서는 Cloudflare Pages 또는 Netlify로 무료 서브도메인 배포 후 반응 확인입니다.
- GitHub Pages와 Vercel Hobby도 가능하지만, 현재 앱은 서버 기능이 필요 없으므로 정적 배포만 사용합니다.
- `_headers`는 Cloudflare Pages/Netlify용 기본 보안·캐시 헤더입니다. GitHub Pages에서는 무시될 수 있습니다.
- `netlify.toml`은 Netlify에서 빌드 없이 현재 폴더를 정적 사이트로 배포하기 위한 설정입니다.
- 배포 전 세부 체크리스트는 `docs/free-web-launch.md`를 확인하세요.

## 실행

정적 파일만으로 동작합니다.

```powershell
python -m http.server 4173 --bind 127.0.0.1
```

브라우저에서 `http://localhost:4173`을 열면 됩니다.

서비스 워커와 홈 화면 추가 기능은 `http://127.0.0.1:4173/`처럼 로컬 서버에서 확인하세요.

## 출시 전 검증

```powershell
node tools/verify-static-launch.mjs
```

이 스크립트는 필수 정적 파일, 법적 안내 링크, 서비스 워커 캐시 버전, 배포 헤더, 무료 웹 베타 문서를 점검합니다.
