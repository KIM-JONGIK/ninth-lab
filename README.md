# 9회말 연구소

실존 선수, 구단 로고, 공식 경기 기록 없이 야구 팬의 감정과 상황을 공유 카드로 만드는 비공식 팬메이드 웹앱입니다.

## 타겟

- 직관 전후에 짧은 문구를 공유하고 싶은 야구 팬
- 팀·선수 실명 없이도 야구 감성을 즐기고 싶은 라이트 팬
- 친구 단톡방, 스토리, 커뮤니티에 올릴 안전한 밈 카드가 필요한 사용자

## 핵심 기능

- 12가지 상황, 4가지 톤, 조합별 5개씩 240개 핵심 문구로 야구 감정 밈 카드 생성
- `한줄 밈`, `단톡 속보`, `상태창`, `관전 일지` 네 가지 밈 문법
- 핵심 문구, 라이브 반응, 일일 소재를 합친 353개 권리 안전 콘텐츠 소스
- 같은 상황·톤의 문구를 한 바퀴 쓰기 전에는 반복하지 않는 세션 문구 덱
- 콘텐츠 풀이 바뀌면 이전 세션 덱을 자동 초기화하는 문구 목록 서명
- 같은 설정의 문구만 바꾸는 `다른 문구`와 전체 설정을 섞는 `랜덤 조합`
- 무권리 그래픽 카드 배경 선택
- 생성 출처, 제약 조건, SHA-256을 고정한 공개 자산 매니페스트
- 데스크톱에서 카드 미리보기를 고정하고 오른쪽 옵션 패널만 스크롤하는 제작 화면
- 첫 방문자가 버튼 하나로 시작하는 `10초 관전 타입` 카드
- 첫 방문자가 설정 없이 시작하는 `바로 1장`, `라이브 짤`, `캡션까지` 버튼
- 모바일 첫 화면에서 설명 없이 바로 누르는 `바로 1장 만들기`, `라이브 짤` 진입 버튼
- 세로로 이어진 문서 대신 한 번에 한 기능만 여는 `만들기·오늘·라이브·팬심·보관함` 앱 탭
- 모바일 제작 화면 안에서 카드와 옵션을 전환하는 `카드 보기·옵션 고르기` 작업 탭
- 커뮤니티에 바로 올리기 쉬운 `라이브 짤판`
- 이 브라우저의 반응 선택과 베이스 샘플을 보여주는 `내 팬심 샘플 %`
- 최근 만든 카드를 이 브라우저에만 저장하는 `보관함`
- 보관함 기본 최근 3장 보기와 전체 보기 토글
- 모바일 공유 시트 또는 카드 상태를 그대로 여는 `공유 링크`
- 커뮤니티 게시글에 붙이기 쉬운 안전 고지 포함 `캡션 복사`
- 서버 없이 질문 링크로 이어지는 `단톡방 반응 릴레이`
- 질문 링크를 받은 사람이 바로 답장 카드 흐름을 이해하는 릴레이 답장 모드
- 미리보기에서 바로 누르는 `친구에게 묻기` 릴레이 진입점
- 채팅에서 답을 받기 쉬운 `단톡방 4지선다` 질문 카드
- 날짜마다 바뀌는 `오늘의 덕아웃` 콘텐츠
- 반복 피로를 줄이는 확장된 일일 문구와 라이브 반응 후보
- 오늘의 한 줄, 단톡방 질문, 안전 공유 소재를 바로 카드로 만드는 원탭 콘텐츠 카드
- 오늘 만든 카드 수, 주간 결산까지 남은 장수, 결산 버튼을 보여주는 `오늘의 1장 챌린지`
- 브라우저 안에만 저장되는 `덕아웃 출석` 연속 일수
- 보관함 카드 3장 이상으로 만드는 `주간 감정 결산`
- 보관함 카드 3장을 경기 전·보는 중·끝난 뒤로 묶는 `3컷 감정 타임라인`
- 삭제한 보관함 카드를 바로 복구하는 `되돌리기`
- 운영자에게 보낼 문구를 만드는 `권리·안전 검토 요청`
- 사용성 의견을 서버 없이 전달하는 `베타 피드백` 문구 복사
- 공개 화면에서는 접혀 있고 필요할 때만 여는 베타 초대·운영 도구
- 로컬에서 열어도 실제 공개 베타 URL을 복사하는 초대 문구
- 브라우저 안의 사용 상태를 요약하는 `로컬 베타 리포트`
- 저장 상태를 보여주고 지울 수 있는 `내 데이터` 관리
- 정적 공개 배포를 위한 `개인정보 안내`와 `이용·권리 기준` 페이지
- 모바일 하단 탭과 PWA 기본 설정
- 화면에서 고른 배경이 반영되는 정사각, 스토리, 가로형 PNG 저장
- 미리보기 아래에서 바로 공유, 링크 복사, PNG 저장
- 선수 대신 일상 루틴을 배치하는 `내 인생 타순`
- 비공식 팬메이드, 창작 상황, 공식 기록 아님 고지

## 운영 원칙

- 실존 선수명, 구단명, 로고, 유니폼, 중계 캡처, 공식 기록을 사용하지 않습니다.
- 앱 콘텐츠는 가상 상황, 팬 감정, 일상 은유, 창작 문구로만 구성합니다.
- 질문 릴레이와 감정 결산은 실제 경기 집계나 공개 리더보드가 아닌 개인 브라우저 기준 카드입니다.
- 상대 팬덤, 선수, 심판을 공격하거나 부상·사생활을 추측하는 표현은 피합니다.
- 권리 침해 요청이 들어오면 해당 콘텐츠를 숨김 처리하고 운영자가 검토하는 흐름을 둡니다.
- 보관함, 반응 카운트, 덕아웃 출석, 문구 순환 상태는 서버 전송 없이 사용자 브라우저 안에만 저장합니다.
- 공개 호스팅 계층의 기술 정보 처리는 앱 자체 로컬 저장과 구분해 개인정보 안내에 표시합니다.
- 권리·삭제 요청은 개인 이메일을 노출하지 않는 공개 GitHub 이슈 양식으로 접수합니다.
- 사용자는 `내 데이터`에서 브라우저 안의 보관함, 반응 카운트, 출석 기록을 지울 수 있습니다.
- 검토 요청 기능은 서버 접수가 아니라 사용자가 운영자에게 전달할 문구를 복사하는 MVP 기능입니다.
- 베타 피드백 기능도 서버 접수가 아니라 사용자가 직접 전달할 문구를 복사하는 MVP 기능입니다.
- 런칭 센터는 실제 배포 계정 상태를 조회하지 않고, 정적 MVP 기준의 운영 준비 목록을 제공합니다.
- 로컬 베타 리포트는 서버 분석이 아니라 해당 브라우저에 남은 카드, 반응, 출석 상태만 요약합니다.
- `legal/privacy.html`은 현재 MVP가 수집하지 않는 정보, 로컬 저장소, 공유 링크, 캐시 동작을 설명합니다.
- `legal/terms.html`은 허용 콘텐츠, 금지 콘텐츠, 팬심 샘플 %의 개인용 성격, 향후 상업 기능 기준을 설명합니다.

## 준비 목록

- 수익화는 당장 제품 화면에 넣지 않습니다.
- 후보 1: 응원용품 제작, 글러브 수선, 사회인야구 장비샵 같은 업체 홍보 탭.
- 후보 2: 야구용품 중고거래는 `장비장터 Beta`로 수요를 확인한 뒤 안전결제와 분쟁 대응 기준이 준비되면 검토.
- 어떤 경우에도 구단 로고, 선수명, 공식 굿즈처럼 보이는 권리물 제작·거래는 허용하지 않습니다.

## 무료 웹 베타

- 이 앱은 정적 파일만으로 동작하므로 무료 정적 호스팅에서 작은 공개 베타를 시작할 수 있습니다.
- 유료 기능을 붙이기 전 주 배포 URL은 `https://kim-jongik.github.io/ninth-lab/`입니다.
- GitHub Pages는 public 저장소와 GitHub Actions 내장 권한만 사용하므로 외부 배포 토큰이 필요 없습니다.
- `main` 푸시는 `.github/workflows/github-pages-deploy.yml`을 통해 검증된 `dist/`만 배포합니다.
- Netlify 자동 배포는 중지했고 기존 워크플로는 수동 비상용으로만 남겨뒀습니다.
- `_headers`는 GitHub Pages에서 적용되지 않지만 향후 Cloudflare Pages 전환과 수동 비상 배포를 위해 유지합니다.
- 광고·업체 홍보·거래 중개를 도입하기 전 Cloudflare Pages 또는 유료 상업 호스팅으로 이전합니다.
- 배포 전 세부 체크리스트는 `docs/free-web-launch.md`를 확인하세요.
- 토큰 없는 GitHub Pages 자동 배포 절차는 `docs/deploy-runbook.md`와 `tools/check-deploy-readiness.mjs`를 사용합니다.

## 실행

정적 파일만으로 동작합니다.

```powershell
python -m http.server 4173 --bind 127.0.0.1
```

브라우저에서 `http://localhost:4173`을 열면 됩니다.

서비스 워커와 홈 화면 추가 기능은 `http://127.0.0.1:4173/`처럼 로컬 서버에서 확인하세요.

## 출시 전 검증

```powershell
node tools/verify-static-launch.mjs
node tools/verify-content-safety.mjs
node tools/verify-phrase-deck.mjs
node tools/verify-asset-provenance.mjs
node tools/check-deploy-readiness.mjs
```

첫 번째 스크립트는 필수 정적 파일, 법적 안내 링크, 서비스 워커 캐시 버전, 배포 헤더, 공개 베타 초대 URL, 공개 배포 빌드, 자동 배포 워크플로, 무료 웹 베타 문서를 점검합니다.
두 번째 스크립트는 Git 원격, GitHub CLI 로그인, public 저장소, GitHub Pages 설정, Pages 자동 배포 및 Netlify 자동 배포 중지 상태를 점검합니다.
